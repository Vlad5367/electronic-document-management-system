# electronic-document-management-system

Веб-приложение на Flask для управления документами: создание, хранение, подписание и отслеживание действий пользователей.

## Основные возможности

### Аутентификация

* Вход в систему по логину и паролю
* Управление пользовательской сессией
* Выход из системы

### Поиск документов

* Поиск по названию
* Фильтрация по дате создания
* Фильтрация по формату файла (PDF, PNG, JPG и др.)

### Работа с документами

* Загрузка документов
* Просмотр документов
* Скачивание файлов
* Добавление примечаний
* Поддержка различных форматов

### Подпись документов

* Добавление подписи (факсимиле)
* Поддержка PDF, PNG, JPG
* Выбор позиции подписи
* Свободное размещение подписи
* При подписании создаётся копия документа

### Создание документов

* Создание документов на основе шаблонов
* Поддержка динамических полей (`{{field}}`)
* Генерация PDF-файлов

### Архив документов

* Перемещение документов в архив
* Просмотр архивных документов
* Полное удаление документов

### Журнал действий

* Логирование действий пользователей:

  * вход и выход из системы
  * просмотр документов
  * загрузка файлов
  * подписание
  * удаление
* Просмотр последних действий

## Технологии

* Backend: Flask
* База данных: PostgreSQL, SQLAlchemy
* Работа с PDF: PyMuPDF (fitz), ReportLab
* Обработка изображений: Pillow
* Frontend: HTML, CSS

## Основные маршруты

| URL              | Описание           |
| ---------------- | ------------------ |
| `/login`         | Вход в систему     |
| `/logout`        | Выход              |
| `/home`          | Главная страница   |
| `/`              | Список документов  |
| `/upload`        | Загрузка документа |
| `/create`        | Создание документа |
| `/document/<id>` | Просмотр документа |
| `/archive`       | Архив              |
| `/logs`          | Журнал действий    |

## Особенности реализации

* Подпись документов:
  * PDF — через PyMuPDF
  * Изображения — через Pillow
* Шаблоны поддерживают динамическую подстановку данных
* Все действия пользователей логируются
* Используется мягкое удаление (`is_deleted`)
